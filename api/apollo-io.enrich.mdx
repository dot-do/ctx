# Apollo.io Organization Enrich API

Enrich company data using domain, name, or LinkedIn URL. Returns comprehensive firmographic data including size, industry, funding, technologies, and social profiles.

## Features

- **60M+ Companies**: Global coverage
- **Rich Data**: Industry, size, revenue, funding
- **Technographics**: Technology stack detection
- **Hierarchies**: Parent companies and subsidiaries
- **Social Profiles**: LinkedIn, Twitter, Facebook, Crunchbase

## Usage

```typescript
export async function enrichOrganization(params: EnrichParams): Promise<EnrichResponse> {
  const response = await fetch('https://api.apollo.io/v1/organizations/enrich', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Api-Key': process.env.APOLLO_API_KEY,
    },
    body: JSON.stringify(params),
  })

  if (!response.ok) {
    throw new Error(`Apollo API error: ${response.status}`)
  }

  return response.json()
}

// Example: Enrich by domain (most common)
const company = await enrichOrganization({
  domain: 'example.com',
})

// Example: Enrich by name
const company2 = await enrichOrganization({
  organization_name: 'Acme Corp',
})

// Example: Enrich by LinkedIn
const company3 = await enrichOrganization({
  linkedin_url: 'https://linkedin.com/company/acme',
})
```

## Best Practices

### Input Strategy
- **Domain** is the most reliable identifier (use when available)
- **Company name** works but may have ambiguity
- **LinkedIn URL** is good for recent companies

### Data Quality
- Employee count is estimated (±20% accuracy)
- Funding data sourced from Crunchbase
- Technologies detected via web scraping
- Update frequently for fast-growing companies

### Caching Strategy
```typescript
// Cache company data for 30-90 days
const cacheKey = `company:${domain}`
const cached = await env.KV.get(cacheKey, 'json')

if (cached) return cached

const fresh = await enrichOrganization({ domain })
await env.KV.put(cacheKey, JSON.stringify(fresh), {
  expirationTtl: 60 * 60 * 24 * 30, // 30 days
})

return fresh
```

### Use Cases
- **Lead Qualification**: Filter by size, industry, funding
- **Account Routing**: Route by geography, industry
- **Personalization**: Use keywords, technologies in outreach
- **Market Research**: Analyze company landscapes

## Data Fields Reference

### Basic Information
- `name`, `website_url`, `primary_domain`
- `logo_url` (from Clearbit or similar)
- `industry`, `keywords` (array)
- `description` (company overview)

### Size & Structure
- `estimated_num_employees` (±20% accuracy)
- `retail_location_count` (physical locations)
- `owned_by_organization_id` (parent company)
- `suborganizations` (subsidiaries)

### Funding & Public Status
- `publicly_traded_symbol`, `publicly_traded_exchange`
- `total_funding` (lifetime funding in USD)
- `latest_funding_round_date`
- `latest_funding_stage` (Seed, Series A/B/C, etc.)

### Contact & Location
- `phone` (main company number)
- `street_address`, `city`, `state`, `postal_code`, `country`

### Online Presence
- `linkedin_url`, `twitter_url`, `facebook_url`
- `crunchbase_url`
- `founded_year`

### Technographics
- `technologies` (array of { category, name } objects)
- Detected from website and public data
- Categories: Languages, Frameworks, Cloud, Analytics, etc.
