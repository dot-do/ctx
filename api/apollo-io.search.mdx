# Apollo.io Person Search API

Search Apollo's database of 275M+ contacts using flexible criteria including job titles, companies, locations, and more.

## Features

- **Flexible Search**: Multiple filter combinations
- **High Match Quality**: 275M+ verified contacts
- **Email Status**: Verified (90-95%), guessed (70-80%), or unavailable
- **Pagination**: Up to 100 results per page
- **Rich Data**: Company info, LinkedIn, social profiles

## Usage

```typescript
export async function searchPeople(params: SearchParams): Promise<SearchResponse> {
  const response = await fetch('https://api.apollo.io/v1/people/search', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Api-Key': process.env.APOLLO_API_KEY,
    },
    body: JSON.stringify(params),
  })

  if (!response.ok) {
    throw new Error(`Apollo API error: ${response.status}`)
  }

  return response.json()
}

// Example search for engineering leaders in tech companies
const results = await searchPeople({
  person_titles: ['VP of Engineering', 'CTO', 'Engineering Director'],
  organization_industries: ['Software', 'Technology'],
  organization_num_employees_ranges: ['51,200', '201,500'],
  page: 1,
  per_page: 50,
})
```

## Best Practices

### Search Optimization
- Use multiple filters to narrow results
- Start with broad criteria, then refine
- Request only the data you need (per_page)
- Cache results to avoid redundant searches

### Email Quality
- Prioritize `verified` email status (90-95% accuracy)
- Use `guessed` emails with caution (70-80% accuracy)
- Always validate before sending emails

### Cost Management
- Each person in results costs 1 credit
- Use pagination wisely (per_page)
- Cache search results for 7-30 days
- Monitor credit usage with Analytics

## Error Handling

```typescript

  if (error.status === 401) {
    // Invalid API key

    // Rate limited - implement retry with backoff
  } else if (error.status === 402) {
    // Insufficient credits
  }
}
```
