# Apollo.io Person Match API

Enrich contact data by matching email, LinkedIn profile, or name + company combination. Returns comprehensive person and company information.

## Features

- **Multiple Identifiers**: Email, LinkedIn, name+company
- **High Accuracy**: Best-in-class match rates
- **Rich Output**: Contact + company data
- **Email Verification**: 90-95% accuracy for verified emails
- **Social Profiles**: LinkedIn, Twitter, GitHub, Facebook

## Usage

```typescript
export async function matchPerson(params: MatchParams): Promise<MatchResponse> {
  const response = await fetch('https://api.apollo.io/v1/people/match', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Api-Key': process.env.APOLLO_API_KEY,
    },
    body: JSON.stringify(params),
  })

  if (!response.ok) {
    throw new Error(`Apollo API error: ${response.status}`)
  }

  return response.json()
}

// Example: Enrich by email
const person = await matchPerson({
  email: 'john@example.com',
  reveal_personal_emails: true,
})

// Example: Enrich by name + company
const person2 = await matchPerson({
  first_name: 'John',
  last_name: 'Doe',
  organization_name: 'Acme Corp',
})

// Example: Enrich by LinkedIn
const person3 = await matchPerson({
  linkedin_url: 'https://linkedin.com/in/johndoe',
})
```

## Best Practices

### Input Strategy
- Provide multiple identifiers for better accuracy
- Email + name/company yields highest confidence
- LinkedIn URL is most reliable single identifier

### Email Quality
- **verified** (90-95% accuracy): Use confidently
- **guessed** (70-80% accuracy): Validate before use
- **unavailable**: No email found

### Caching
- Cache enriched data for 30-90 days
- Use KV storage with email/LinkedIn as key
- Reduces API costs by ~90%

### Privacy Compliance
- Only use `reveal_personal_emails: true` when legally permitted
- Respect unsubscribe requests
- Follow GDPR/CCPA guidelines

## Data Fields Reference

### Person Fields
- `first_name`, `last_name`, `email`, `email_status`
- `title`, `seniority` (entry, mid, senior, executive, owner)
- `linkedin_url`, `twitter_url`, `github_url`, `facebook_url`
- `photo_url` (profile picture)
- `city`, `state`, `country` (location)
- `phone_numbers` (array of phone objects)

### Company Fields (nested in organization)
- `name`, `website_url`, `primary_domain`
- `industry`, `keywords` (array of technology/business keywords)
- `estimated_num_employees`
- `logo_url`, `linkedin_url`, `twitter_url`, `facebook_url`
- `phone` (main company number)
